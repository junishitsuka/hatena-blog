# CLAUDE.md

このファイルは、Claude Code (claude.ai/code) がこのリポジトリで作業する際のガイダンスを提供します。

## 会話ガイドライン

常に日本語で回答してください。

## リポジトリ概要

junishitsukaのはてなブログ記事管理リポジトリです。AIアシスタント（Claude Code）を用いてブログ記事の管理と執筆を行います。記事ははてなブログ固有のフロントマターと記法を使用したMarkdown形式で記述されます。

## ディレクトリ構成

- `blogs/`: ブログドメインごとに整理された公開済み記事
  - `mitsuruya.hatenablog.com/`: 個人ブログ（ネタは何でもあり）
  - `product.plex.co.jp/`: 株式会社プレックスのテックブログ
- `guidelines/`: 一貫したスタイルを維持するための包括的な執筆ガイドライン
  - `writing-style.md`: junishitsukaの執筆スタイルと人格の分析
  - `article-categories.md`: 記事の分類（技術記事、組織記事、趣味記事）
  - `article-templates.md`: 記事タイプ別のテンプレート
  - `ai-prompts.md`: AI支援執筆のためのプロンプト例
- `outlines/`: 執筆予定の記事のアウトライン

## 記事の執筆

### 執筆の基本原則

junishitsukaとして記事を執筆する際は、以下の原則に従ってください：

1. **一人称**: 常に「自分」を使用し、「私」は使わない
2. **文体**: 丁寧語（です・ます調）、謙虚で読者に配慮した表現
3. **構成**: 箇条書きと具体例を含む明確なセクション分け
4. **誠実さ**: 失敗や課題を隠さず率直に共有
5. **絵文字なし**: 仕事関連の記事（技術記事・組織記事）では使用しない

### 標準的な記事構成

```markdown
---
title: [記事タイトル]
published: YYYY-MM-DD
updated: YYYY-MM-DDTHH:MM:SS+09:00
url: https://[domain]/entry/[slug]
tags:
  - Tag1
  - Tag2
---

[f:id:xxx:image]

こんにちは、プレックスの[石塚](https://twitter.com/ij_spitz)です。

[1-2段落: 背景と動機]

[:contents]

# セクション1
## サブセクション1-1
...

# おわりに / 終わりに
[まとめと今後の展望]

[技術記事の場合は採用情報リンク]
```

### 記事カテゴリ

**技術記事** (product.plex.co.jp):
- 実装の詳細、Tips、運用ノウハウに焦点
- 実装前後の比較を含むコード例を掲載
- 採用情報で締める
- 例: 「オンプレRedashの運用Tips8選」

**組織記事** (product.plex.co.jp):
- チームビルディング、イベント、採用
- 構成: 背景（2つの理由）→ 実施内容 → 振り返り
- 画像と参加者の声を含める

**通常記事** (mitsuruya.hatenablog.com):
- 趣味（ワイン、日本酒、グルメ）、キャリア、生産性
- フォーマット: 「X選」や「体験記」
- 採用情報リンクは不要

### はてなブログ記法

- リンク: `[https://example.com/:embed:cite]`
- 画像: `[f:id:username:imageid]`
- 目次: `[:contents]`
- コードブロック: 言語名を指定（```ruby, ```sql など）

### 頻出フレーズ

**導入**:
- 「こんにちは、プレックスの[石塚](https://twitter.com/ij_spitz)です。」
- 「今回の記事では〜について紹介します。」

**謙虚な表現**:
- 「個人的に〜と感じています。」
- 「〜という方針を取っています。」
- 「少しでも参考になれば幸いです。」
- 「手前味噌ではありますが〜」
- 「恥ずかしながら〜」

**今後の行動**:
- 「〜していきます。」
- 「〜していきたいと思います。」

### 執筆の主要な特徴

1. **謙虚さ**: 断定的な表現を避け、「個人的には」や「〜と感じています」のような限定表現を使用
2. **具体性**: 常に具体例、数字、コードを提供
3. **透明性**: 成功も失敗も正直に共有
4. **読者への配慮**: 背景を丁寧に説明し、参考リンクを提供
5. **進行形**: 「〜していきます」を頻繁に使用
6. **箇条書き**: 複雑な情報はリストで整理
7. **トレードオフ**: 複数のアプローチとその長所・短所を認識
