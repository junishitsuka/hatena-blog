# 記事タイプ別の執筆テンプレート

記事のカテゴリごとに、効果的な記事構成のテンプレートを提供します。

## 技術記事テンプレート

技術的な実装、Tips、運用ノウハウを共有する記事に使用します。

```markdown
# [技術要素]を[動詞]した話 / [技術要素]の[Tips/実装方法]

[アイキャッチ画像]

こんにちは、プレックスの[石塚](https://twitter.com/ij_spitz)です。

[1-2段落: 背景や問題意識]
今回の記事では、[技術要素]について、[目的]をするために[やったこと]を紹介します。

[:contents]

# [技術要素]とは？
[技術の概要を2-3段落で説明]

# 課題/背景
[導入の理由や直面していた課題を説明]

以下のような課題がありました：
- 課題1
- 課題2
- 課題3

# 実装/改善内容

## 1. [項目1]
[説明文]

**実装前のコード**
```[言語]
[コード例]
```

**改善後のコード**
```[言語]
[コード例]
```

## 2. [項目2]
...

# 振り返り/学び
[実際にやってみての感想、学んだこと]

# おわりに
今回は[まとめ]について紹介しました。少しでも参考になれば幸いです。

プレックスでは[職種]を募集しています。
少しでも興味を持っていただけた方は業務委託や副業からでも、ぜひご応募いただけると幸いです。

[https://dev.plex.co.jp/:embed:cite]
```

### ポイント
- タイトルは「〜した話」「〜のTips」など具体的に
- 技術の概要説明から始めて、読者の理解を助ける
- 課題→解決策→振り返りの流れで構成
- コードは実装前後を比較して示す
- 最後に採用情報を必ず入れる

## 組織・イベント記事テンプレート

組織運営、イベントレポート、採用に関する記事に使用します。

```markdown
# [イベント名/取り組み名]

[アイキャッチ画像]

こんにちは、プレックスの[石塚](https://twitter.com/ij_spitz)です。

[イベント/取り組みの概要を1-2段落で紹介]

[:contents]

# 背景
[なぜこの取り組みを始めたのか、2つの理由を説明]

1つ目は[理由1]です。[詳細説明]

2つ目が[理由2]です。[詳細説明]

# 実施内容

## [項目1]
[具体的な内容を説明、必要に応じて画像を挿入]

<figure class="figure-image figure-image-fotolife" title="説明">[画像]<figcaption>説明</figcaption></figure>

# まとめ/今後について
[振り返りと今後の展望]

# 終わりに
[締めのメッセージと採用情報]
```

### ポイント
- 背景は「1つ目」「2つ目」と明確に分けて説明
- 実施内容では画像を積極的に使用
- 今後の展望や継続性を示す
- イベントの場合は参加者の声や雰囲気を伝える

## 趣味・ライフスタイル記事テンプレート

個人的な趣味、体験、ライフハックなどを共有する記事に使用します。

```markdown
# [テーマ][数]選

[前置き: なぜこの記事を書くのか]

[:contents]

# [テーマ][数]選

[概要説明]

## 1. [項目1]
[個人的な感想と具体的な説明]

## 2. [項目2]
...

# 終わりに
[まとめと今後の展望]
```

### ポイント
- 「〜選」「〜体験記」など親しみやすいタイトル
- 前置きで執筆の動機を簡潔に説明
- 各項目は個人的な感想を中心に
- 具体的な情報（価格、場所、スペックなど）も含める
- 採用情報は不要

---

## テンプレート使用時の注意点

### 共通事項
- テンプレートは参考として使用し、内容に応じて柔軟にカスタマイズ
- 読者が求める情報を最優先に考える
- 無理に型に当てはめず、自然な流れを意識

### カスタマイズのポイント
- セクションの追加・削除は自由
- 順序の入れ替えも内容に応じて可能
- 重要なのは「読みやすさ」と「情報の伝わりやすさ」

### 避けるべきパターン
- テンプレートの形式的な適用（中身が薄い）
- 無理な箇条書きや番号付け
- テンプレートに合わせた不自然な構成
